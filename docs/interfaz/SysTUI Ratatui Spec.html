SysTUI on Ratatui design spec · v0.1 · ratatui 0.29 · tokio 1.40 · russh 0.45 · rust edition 2024

This document maps the visual prototype to a concrete Rust + Ratatui implementation: how state is modelled, how SSH and probes are isolated from the render loop, how the check engine generates the health score, and how each screen renders.

10 screens ~15K SLOC est. 1 render thread N async tasks ~120 hrs to MVP

01 Goals & non-goals

In scope (MVP)

  • Inventory of SSH hosts with tags, health, fast jump (1 round-trip to render dashboard).
  • Read-only inspection of system: services, processes, sockets, disks, logs, crons, containers.
  • Curated actions behind confirmation, audit log, automatic backup of edited files.
  • Health score from a deterministic, explainable check engine — no LLM, no remote telemetry.
  • Local-only operation: ~/.config/systui/ for state, ~/.local/share/systui/ for audit and backups.

Out of scope (defer or never)

  • Editing arbitrary remote files in-place (use $EDITOR on the host via SSH instead).
  • Multi-tenant / RBAC. One user, one machine. SSH does the auth.
  • Time-series storage beyond the in-memory ring buffer (delegate to Prometheus/Grafana).
  • Pushing changes back to config management (Ansible/Salt). Out.

03 Architecture

Single render thread, many async workers. Components communicate over typed channels. The render loop is purely a function of Model and never blocks on I/O.

Three message types flow:

  • Msg — user input or async result; consumed by update.
  • Cmd — side-effect request emitted by update (e.g. "fetch services on host X"); consumed by the dispatcher.
  • Event — raw input from crossterm or tick from a 250ms interval.

05 Elm-style state

One Model, one update, one view. The update function is pure: it can never call tokio::spawn, never hit the network, never touch the terminal. Side effects are expressed as Cmd values returned alongside the new model.

// src/update.rs
pub fn update(mut m: Model, msg: Msg) -> (Model, Vec<Cmd>) {
    use Msg::*;
    match msg {
        Tick => {
            m.now = Instant::now();
            let mut cmds = vec![];
            if let Some(h) = m.attached_host() {
                if m.probes.is_stale(h, Probe::Metrics) {
                    cmds.push(Cmd::Probe(h, Probe::Metrics));
                }
            }
            (m, cmds)
        }

        Key(k) => handle_key(m, k),

        OpenHost(id) => {
            m.attached = Some(id);
            m.tab = Tab::Dashboard;
            (m, vec![
                Cmd::Dial(id),
                Cmd::Probe(id, Probe::Identity),
                Cmd::Probe(id, Probe::Metrics),
                Cmd::RunChecks(id),
            ])
        }

        ProbeResult(id, Probe::Services, Ok(units)) => {
            m.hosts.get_mut(&id).map(|h| h.services = units);
            (m, vec![])
        }

        Action(req) if m.is_read_only(req.host) => {
            m.toasts.push(Toast::blocked("host is read-only"));
            (m, vec![])
        }
        Action(req) => {
            m.pending_action = Some(req);
            (m, vec![])
        }
        ActionConfirm => {
            let req = m.pending_action.take().unwrap();
            (m, vec![Cmd::ExecAction(req)])
        }

        // …
    }
}

The runtime turns each Cmd into a tokio::spawn; the spawned task ultimately sends a Msg back through the channel. This is the only way to mutate state.

// src/cmd.rs
pub enum Cmd {

07 Event loop

One loop. Three sources: crossterm events, the message channel, and a 250ms tick. The render call is non-blocking and happens at most once per loop iteration.

// src/app.rs
pub async fn run(mut term: Terminal<CrosstermBackend<Stdout>>) -> Result<()> {
    let (msg_tx, mut msg_rx) = mpsc::channel::<Msg>(512);
    let dispatcher = Dispatcher::spawn(msg_tx.clone()).await?;
    let mut model = Model::bootstrap().await?;

    let mut events = crossterm_event_stream();
    let mut tick = tokio::time::interval(Duration::from_millis(250));

    loop {
        term.draw(|f| ui::view(f, &model))?;

        let msg = tokio::select! {
            ev = events.next() => match ev {
                Some(Ok(e)) => Msg::from_event(e),
                _ => continue,
            },
            m = msg_rx.recv() => match m {
                Some(m) => m,
                None => break,
            },
            _ = tick.tick() => Msg::Tick,
        };

        if matches!(msg, Msg::Quit) { break; }
        let (next, cmds) = update::update(model, msg);
        model = next;
        for c in cmds { dispatcher.submit(c).await; }
    }
    Ok(())
}

08 SSH session pool

One Session per host, kept alive. Each session multiplexes channels: short exec channels for probes, long-running channels for log tails. The pool caps connections and evicts idle sessions.

// src/ssh/pool.rs
pub struct Pool {
    sessions: Mutex<HashMap<HostId, Arc<Session>>>,
    cap: usize,
}

impl Pool {
    pub async fn get_or_dial(&self, h: &Host) -> Result<Arc<Session>> {
        if let Some(s) = self.sessions.lock().await.get(&h.id) {
            return Ok(s.clone());
        }
        let s = Session::dial(h).await?;
        let arc = Arc::new(s);
        self.sessions.lock().await.insert(h.id.clone(), arc.clone());
        Ok(arc)
    }
}

// src/ssh/exec.rs — typed exec
pub trait FromStdout: Sized {
    fn parse(out: &str) -> Result<Self>;
}

impl Session {
    pub async fn run<T: FromStdout>(&self, cmd: &str) -> Result<T> {
        let mut ch = self.channel_open_session().await?;
        ch.exec(true, cmd).await?;
        let out = ch.collect_stdout(DEFAULT_TIMEOUT).await?;
        T::parse(&out)
    }
}

// Example: parsing `ss -tlnpJ`
impl FromStdout for Vec<Listener> {
    fn parse(out: &str) -> Result<Self> {
        let v: SsJsonRoot = serde_json::from_str(out)?;
        Ok(v.into_listeners())
    }
}
Pitfall. Don't open a new SSH connection per probe. Cold connect to a remote host is 200–800ms. With 10 probes that's 2–8 seconds before anything renders. Always reuse the session.

09 Probes & cache

// src/probes/mod.rs
pub struct ProbeCache {
    entries: HashMap<(HostId, Probe), Entry>,
}
struct Entry {
    fetched: Instant,
    inflight: bool,
}
impl ProbeCache {
    pub fn should_fetch(&self, h: &HostId, p: Probe, now: Instant) -> bool {
        match self.entries.get(&(h.clone(), p)) {
            None => true,
            Some(e) => !e.inflight && now.duration_since(e.fetched) > p.ttl(),
        }
    }
}

10 Check engine

Each check is independent, deterministic, takes a snapshot of host state, returns zero or more Findings. The health score is just 100 + sum(finding.score_delta), floored at 0. Everything the user sees on the Security and Dashboard tabs is generated here, including the "explain" breakdown.

// src/checks/mod.rs
pub trait Check: Send + Sync {
    fn id(&self) -> &static str;
    fn module(&self) -> Module;
    fn requires(&self) -> &[Probe];
    fn evaluate(&self, h: &Host) -> Vec<Finding>;
}

// src/checks/disk_pressure.rs
pub struct DiskPressure;

impl Check for DiskPressure {
    fn id(&self) -> &static str { "disk.pressure" }
    fn module(&self) -> Module { Module::Disk }
    fn requires(&self) -> &[Probe] { &[Probe::Disks] }

    fn evaluate(&self, h: &Host) -> Vec<Finding> {
        let Some(snap) = &h.metrics else { return vec![] };
        snap.disks.iter().filter_map(|d| match d.used_pct {
            p if p >= 90  => Some(finding(Severity::Critical, -15, d)),
            p if p >= 80  => Some(finding(Severity::High,     -10, d)),
            p if p >= 70  => Some(finding(Severity::Medium,   -3,  d)),
            _              => None,
        }).collect()
    }
}

11 Log streaming

Tailing logs through SSH is the most expensive thing in the app. Strategy: one tail task per source, decoupled from the render loop, with a bounded ring buffer.

// src/logs/stream.rs
pub fn spawn_tail(
    sess: Arc<Session>,
    src: LogSource,
    out: mpsc::Sender<Msg>,
) -> TailHandle {
    let tail_id = TailId::new();
    let (abort_tx, mut abort_rx) = oneshot::channel();
    tokio::spawn(async move {
        let mut ch = sess.exec_stream(&src.cmd()).await?;
        let mut buf = BytesMut::with_capacity(64 * 1024);

        loop {
            tokio::select! {
                _ = &mut abort_rx => break,
                read = ch.read_buf(&mut buf) => {
                    if read? == 0 { break; }
                    while let Some(line) = extract_line(&mut buf) {
                        let parsed = LogLine::parse(line, src);
                        out.send(Msg::LogAppend(tail_id, parsed)).await.ok();
                    }
                }
            }
        }
        out.send(Msg::LogTailEnded(tail_id)).await.ok();
        Ok::<_, eyre::Error>(())
    });
    TailHandle { id: tail_id, abort: abort_tx }
}

The receiving end uses a fixed-size ring buffer per tail so memory stays bounded even when the user pauses follow. Fingerprints are recomputed on a debounced timer, never per line.

12 Actions & audit

Every state-changing operation is an Action. The model gates them in a single place: if the host is read-only or the action's risk demands a confirmation phrase, the dispatcher cannot receive an ExecAction command at all.

// src/actions/mod.rs
pub struct ActionReq {
    pub host: HostId,
    pub kind: ActionKind,
    pub requires_phrase: Option<String>,
}

pub enum ActionKind {
    UnitRestart(String),
    UnitReload(String),
    UnitStop(String),
    UnitDisable(String),
    ProcessSignal { pid: u32, signal: Signal },
    CronEdit { file: PathBuf, new_body: Vec<u8> },
    SecurityFix { finding: FindingId },
}

impl ActionKind {
    pub fn risk(&self) -> Risk {
        use ActionKind::*;
        match self {
            UnitReload(_)                => Risk::Low,
            UnitRestart(_) | CronEdit{..}  => Risk::Medium,
            UnitStop(_) | UnitDisable(_) => Risk::High,
            ProcessSignal{ signal, .. } if *signal == Signal::KILL => Risk::High,
            ProcessSignal{..}             => Risk::Medium,
            SecurityFix{..}                => Risk::High,
        }
    }
}

// src/actions/audit.rs — append-only, fsync each entry
pub struct AuditEntry {
    pub ts: DateTime<Utc>,
    pub actor: String,           // $USER@$HOSTNAME
    pub host: HostId,
    pub kind: ActionKind,
    pub outcome: Outcome,        // Ok | Failed(stderr) | Blocked(reason)
    pub latency_ms: u32,
}
Backup before edit. Any action that mutates a file (cron edit, sshd_config patch) writes the original to ~/.local/share/systui/backups/<host>/<path>-<ts>.bak first. The audit entry includes the backup path.

13 Render strategy

Ratatui re-renders the full frame every loop. That's fine — at 60Hz a 200×52 frame is ~1.5ms in pure stack-allocated buffers. Don't fight it; never cache widgets, never partial-render. Memoize only expensive computations (fingerprinting, syntax-highlighting), never UI primitives.

// src/ui/mod.rs
pub fn view(f: &mut Frame, m: &Model) {
    let [top, tabs, body, status] = Layout::vertical([
        Constraint::Length(3),
        Constraint::Length(1),
        Constraint::Min(0),
        Constraint::Length(1),
    ]).areas(f.area());

    chrome::top_bar(f, top, m);
    chrome::tabs(f, tabs, m);
    match m.tab {
        Tab::Hosts     => screens::hosts::render(f, body, m),
        Tab::Dashboard => screens::dashboard::render(f, body, m),
        Tab::Services  => screens::services::render(f, body, m),
        // …
    }
    chrome::status_bar(f, status, m);

    // Overlays render last, drawn on top of body.
    if let Some(p) = &m.palette        { overlay::palette::render(f, p, m); }
    if let Some(a) = &m.pending_action { overlay::confirm::render(f, a, m); }
    overlay::toasts::render(f, &m.toasts);
}

// scoring: nucleo (fzf algorithm)
pub fn rerank(items: &mut Vec<Scored>, q: &str) {
    let mut matcher = Matcher::new(Config::DEFAULT);
    items.iter_mut().for_each(|s| s.score = matcher.fuzzy_match(&s.text, q));
    items.sort_by_key(|s| Reverse(s.score));
}

20 Risks & open questions

Risks

  • SSH parsing brittleness. systemctl, ss, docker output drift across distros. Pin to --output=json wherever it exists; for everything else, parse defensively and surface "unknown format" as a Finding rather than panicking.
  • Terminal capability detection. Truecolor isn't universal — fall back gracefully to 256-color via colorterm detection; never assume Unicode beyond box-drawing + the 8 spark glyphs.
  • Long-running SSH and reconnects. Wireguard naps, laptops sleep. The session must surface "lost connection · retry?" inline on the host chip, not crash.
  • Mouse on tmux. Crossterm mouse works but tmux passthrough varies. Keep all flows keyboard-first.

Open questions

  • Plugin model? External binary that emits Findings as JSONL? Defer to v0.2.
  • Multi-host fanout? Run a check across all "prod" hosts at once? Useful, but adds a fleet view we haven't designed yet.
  • Persistent ring of historical metrics beyond the in-memory buffer? Probably a ~/.local/share/systui/history.db with sqlite. Defer.
  • Secrets in SSH commands. Some probes need sudo. Today we assume passwordless sudo or root; full askpass integration is a separate spike.
